How do I digitally archive myself?

Details

Deconstructing the design of this website

Role

Self-directed

Tools

HTML, CSS, JS & p5.js

Need

By the time I graduated in 2022, I understood that there was a need for me to document & archive my work. I had three reasons: (a) to easily share work with people, (b) to have a digital archive of my journey, and (c) to better understand my process by writing about it.


Process

Throughout 2022-2023, I read religiously for ~30 minutes everyday. This could include papers, books, blogs, articles and pretty much anything that tickled my fancy. By doing this, I came across several personal websites that acted as inspiration for my own. Here are all the websites that (I think) influenced my design of this portfolio:

To be in full control of my space on the internet, I developed my website from scratch by using HTML, CSS & JavaScript. Fun fact: I’d never developed a website before.

Design Decisions

I wanted the website to change based on the width of the device that it was being viewed on. The entire website was, therefore, built to be responsive (mostly by using CSS flexbox). Furthermore, type sizes are fluid: dependent on the width of the device.


Type sizes changing with the device's width. Tool used: Utopia Fluid Type Scale Calculator.

Since a lot of the content would be words, careful consideration was given to the typography. Atkinson Hyperlegible is used for longform content since the typeface was developed specifically to improve comprehension. Victor Mono is the second typeface that is used, to include a programming font & add a little bit of flair with the unique italic style design.

A screenshot of an initial typography test.

Each page has been designed with strong intent. The landing page is meant to be a clear digital brain, for people to choose a section of it to go further. If they do not interact with the page (by moving the mouse), the centre text breaks into free-moving particles. This was inspired by Yehwan Song’s website, where no user interaction leads to a fly coming on the website.


Once people select the section they want to go to, everything was designed to be geared towards their search. For example, the work page includes cards with different coloured tags. This was done for people to scroll through the archive (thereby seeing the volume of work) and identify what type of projects they’d want to look at.


Another example is the case study page. It was primarily built for scanning (by including section heads that break the grid), as well as for comfortable long-form reading (by using comfortable leading, line length & big font sizes on wider screens).
